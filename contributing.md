# Contribution Guidelines

Please note that this project is released with a [Contributor Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). By participating in this project you agree to abide by its terms.

## Adding a new tool

To add a new AI coding agent or tool, please submit a pull request with the following:

- **Name**: The official name of the tool.
- **URL**: A link to the tool's homepage or GitHub repository.
- **Description**: A concise, one-line description ending with a period.
- **Category**: The most appropriate category from the existing list.
- **Tags**: Indicate if the tool is `Open Source`, `Free`, `Freemium`, or `Paid`.

### Format

Each entry should follow this format:

```markdown
- [Tool Name](https://example.com) - One-line description ending with a period. `Tag1` `Tag2`
```

### Rules

- Search previous suggestions before making a new one to avoid duplicates.
- Make an individual pull request for each suggestion.
- New categories or improvements to existing categorization are welcome.
- Keep descriptions short, simple, and to the point.
- Descriptions should not be promotional or marketing copy.
- Start the description with a capital letter and end with a period.
- Check your spelling and grammar.
- Entries within each category must be sorted alphabetically.
- The tool must be functional, actively maintained, and have a public presence.
- The pull request title should follow the format: `Add [Tool Name]`.

### Quality standards

- The tool should have been available for at least 30 days.
- The tool should have clear documentation or a landing page.
- The tool should be relevant to AI-assisted software development.

## Updating an existing entry

If a tool has changed its name, URL, or description, please submit a pull request with the updated information and a brief note explaining the change.

Thank you for your suggestions!
